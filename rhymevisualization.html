Designed and programmed visualization tool that allows users to view node-link diagrams of a poet’s rhymed words in their corpus, grouped by phoneme

Repo Link to site

Main Page

Concept: There are very few pieces of software that visualize poetry, presenting many exciting opportunities in data visualization for rhyme. Through this visualization, users can immediately comprehend a poet’s most common phoneme ending, instances of imperfect rhyme, and the connections between words in context throughout a poet’s corpus. A user can also click on a phoneme group to view the visualization in detail, displaying both node (rhymed word) and link (rhyming lines). In this project, I’ve analyzed two poet datasets—Shakespeare’s sonnets and Alexander Pope’s couplets. Due to the difference in time period and type of poetry, I believed these initial datasets would effectively display how these rhyme visualizations can vary widely across datasets.

Creating this tool required data cleaning using python scripts and was ultimately implemented in D3.js, a data visualization software.

What I learned: Data cleaning with Python, node-link visualizations with D3
